# Execution Stages

This document operationalizes the five-stage delivery strategy defined in the product plan.

## Stage 1 - Foundational

- Build workspace shell, ontology contracts, and idea intake.
- Success criteria:
  - All core domain schemas pass validation.
  - Navigation and project context are stable.
  - Readiness API accepts typed contracts.
- Exit gate:
  - `npm run check`, `npm run lint`, and `npm run test` all pass.

## Stage 2 - Product Core

- Deliver full artifact workflow:
  - Blueprint Studio
  - Readiness Screen
  - Roadmap / Milestones
  - Decision Journal
- Success criteria:
  - Users can complete the full blueprint -> readiness -> sequence workflow.
  - Essential and advanced progressive disclosure modes are functional.
- Exit gate:
  - No dead-end routes in primary workflow.

## Stage 3 - Intelligence

- Harden hybrid reasoning stack:
  - Deterministic validation
  - Archetype pattern inference
  - Explainable recommendation output
- Success criteria:
  - Every recommendation has explainability trace.
  - Scope reduction recommendations appear for overreach patterns.
- Exit gate:
  - Intelligence tests cover readiness and overreach pathways.

## Stage 4 - Growth and Reflection

- Expand longitudinal builder growth:
  - Reflection delta reporting
  - Decision quality trends
  - Next-project calibration
- Success criteria:
  - Users can compare planned vs actual outcomes.
  - System emits growth signals that inform next blueprint.
- Exit gate:
  - Reflection artifacts are tied to project timelines.

## Stage 5 - Refinement and Scale

- Add ecosystem capabilities:
  - Template expansion
  - Collaboration model
  - Analytics and observability enrichment
- Success criteria:
  - Feature expansion does not break ontology contracts.
  - Performance and recommendation quality remain stable.
- Exit gate:
  - Architecture review confirms maintainability and bounded complexity.
